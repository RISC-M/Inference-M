### **Source Files**
* **`compile.py`**: A Python script that utilizes Apache TVM to generate optimized hardware-specific mathematical kernels for Transformer workloads (Matrix Multiplication, Residual Add, and ReLU). It outputs RISC-V Vector (RVV) assembly.
* **`profiler.py`**: A custom workload characterization script. It parses the generated assembly files to extract instruction mixes, vector state configurations (`vsetvli`), and register pressure, giving us a blueprint for our hardware microarchitecture.
* **`main.c`**: The primary C wrapper. It initializes the data arrays, calls the TVM-generated kernels, and handles the output or verification of the results.
* **`start.S`**: Bare-metal startup assembly code. It sets up the CPU state (like initializing the stack pointer) before jumping into the `main()` function in C.
* **`link.ld`**: The Linker Script. It defines the memory layout of the target RISC-V hardware, dictating exactly where the compiler should place the code (`.text`) and data (`.data`/`.bss`) in memory.
* **`build.sh`**: The master build orchestrator. It sets up the necessary environment variables, executes `compile.py` to get the assembly, and then runs the RISC-V GCC cross-compiler to link everything into a final executable.

### **Output Files (Generated during build)**
* **`tvm_*.s` files** (e.g., `tvm_matmul.s`, `tvm_residual_add.s`, `tvm_relu.s`): The raw, optimized RISC-V assembly code generated by `compile.py`.
* **`kernel.elf`**: The final Executable and Linkable Format (ELF) binary. This statically-linked file is the final product, ready to be executed on a RISC-V simulator (like Spike or QEMU) or flashed onto actual silicon. *(Note: Do not commit this to Git)*.

## Environment Setup Instructions

### **Step 1: Clone the Repository**
```bash
# Clone the project and navigate into the root directory
git clone git@github.com:RISC-M/Inference-M.git
cd Inference-M
```

### **Step 2: Download Apache TVM Source**
We must compile TVM from source locally to enable the LLVM backend required for RISC-V vector generation.
```bash
# Clone TVM into the root of the project (ignored by git)
git clone --recursive [https://github.com/apache/tvm](https://github.com/apache/tvm) tvm_src
```

### **Step 3: Setup the Python Virtual Environment**
Create an isolated environment so we don't conflict with system packages.
```bash
# Create the environment named 'rvv_tvm_env'
python3 -m venv rvv_tvm_env

# Activate it
source rvv_tvm_env/bin/activate

# Install required base dependencies
pip install --upgrade pip
pip install numpy psutil typing_extensions apache-tvm-ffi
```

### **Step 4: Build the TVM C++ Core**
We need to compile TVM with LLVM enabled to support cross-compiling to our RISC-V target.
```bash
cd tvm_src
mkdir build
cd build

# Copy the default config
cp ../cmake/config.cmake .

# Enable LLVM (Mandatory for our pipeline)
echo "set(USE_LLVM ON)" >> config.cmake

# Configure and compile (This takes 10-20 minutes depending on CPU)
cmake ..
make -j$(nproc)
```

### **Step 5: Link TVM via Environment Variables (Recommended)**
Instead of installing TVM into the Python virtual environment via `pip`, we will dynamically link the source code and the compiled C++ binaries using environment variables. This gives us strict control over the build paths.

You must add these variables to your environment before running any TVM scripts. The best way to handle this is to place them at the very top of your `compiler/build.sh` script:

```bash
# 1. Define the absolute path to the TVM source
export TVM_HOME=$(realpath ../tvm_src)

# 2. Point Python to the TVM python code
export PYTHONPATH=$TVM_HOME/python:${PYTHONPATH}

# 3. Point Python to the compiled C++ binaries (.so files)
export TVM_LIBRARY_PATH=$TVM_HOME/build/lib

# 4. Point the OS Linker to the compiled binaries (Safety net)
export LD_LIBRARY_PATH=$TVM_HOME/build/lib:${LD_LIBRARY_PATH}
```

## Running the Compiler Pipeline

Once your environment is set up, running the actual pipeline is just one command. Ensure you are in the `compiler` directory.

```bash
./build.sh
```

**Expected Output:**
If everything is configured correctly, the script will output `Environment set...`, generate the assembly files (`tvm_matmul.s`, `tvm_residual_add.s`, `tvm_relu.s`), compile the C and Assembly files, and successfully link them into your final `kernel.elf` binary!

## Profiling the Assembly (Workload Characterization)

To design our microarchitecture efficiently, we need to know exactly which RISC-V Vector instructions our software actually uses. We use `profiler.py` to aggregate this data.

After running `./build.sh`, stay in the `compiler/` directory and run the profiler against all generated assembly files using the `*.s` wildcard:

```bash
python profiler.py *.s
```

**What the Profiler tells you:**
1. **Instruction Mix:** Separates Scalar and Vector operations so you know which execution units (Multipliers, Adders, Memory Loaders) are most critical.
2. **Vector State (vsetvli):** Reveals the required Standard Element Width (SEW) and Length Multipliers (LMUL) needed for the datapath.
3. **Register Pressure:** Shows how many physical vector registers the compiler actually utilized, allowing us to optimize the size of our hardware register file.
